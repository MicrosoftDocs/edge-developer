## CSS features

The following new Cascading Style Sheets (CSS) features are included in Microsoft Edge.



<!-- ------------------------------ -->
#### `border-shape`

The `border-shape` property lets you create non-rectangular borders using any arbitrary shape, such as polygons, circles, or custom paths.

Unlike `clip-path`, which clips the entire element, `border-shape` defines the shape of the border itself, decorates it, and only clips the element's content.  You can either stroke a shape or fill between two shapes.

See also:
* [Border Shaping](https://drafts.csswg.org/css-borders-4/#border-shape) in _CSS Borders and Box Decorations Module Level 4_.


<!-- ------------------------------ -->
#### `contrast-color()`

The `contrast-color()` function automatically returns either black or white, depending on which provides the highest contrast against a given color.

Use this function to meet accessibility contrast requirements easily.  You can use `contrast-color()` anywhere in CSS where a color value is expected.

See also:
* [contrast-color()](https://developer.mozilla.org/docs/Web/CSS/Reference/Values/color_value/contrast-color) at MDN.


<!-- ------------------------------ -->
#### CSS update: decoupling of Width and Style properties

The behavior of `border-width`, `outline-width`, and `column-rule-width` properties has been updated to align with the latest CSS specifications.

Previously, if the corresponding `border-style`, `outline-style`, or `column-rule-style` property was set to `none` or `hidden`, the computed width would be forced to `0px`, regardless of the value that you specified.

Now, the computed values of `border-width`, `outline-width`, and `column-rule-width` always reflect your specified values, independent of the corresponding `*-style` property. The resolved values that are returned by `getComputedStyle()` for `outline-width` and `column-rule-width` also reflect the values that you specified.

See also:
* [border-width](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/border-width) at MDN.
* [outline-width](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/outline-width) at MDN.
* [column-rule-width](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/column-rule-width) at MDN.


<!-- ------------------------------ -->
#### Clip text overflow at user interaction

When you interact with text that has `text-overflow: ellipsis` set, the text temporarily switches from ellipsis to clip mode, allowing you to see and interact with the hidden overflow content.

This feature works for all editable and non-editable elements.  For form controls such as `<textarea>` and `<input>`, this behavior is already supported.

See also:
* [text-overflow](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/text-overflow) at MDN.


<!-- ------------------------------ -->
#### Timeline named range `"scroll"`

In scroll-driven animations, view timelines now support a `"scroll"` named range, in addition to the existing named ranges (`"entry"`, `"exit"`, `"cover"`, `"contain"`).

The new `"scroll"` range refers to the full extent of the scroll container, rather than only the portion of the scroll container in which the animation subject is visible.  This gives you more control over when animations start and stop within the scrollable area.

See also:
* [animation-range](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/animation-range) at MDN.


<!-- ------------------------------ -->
#### Web app scope system accent color

The `accent-color: auto` CSS property is now scoped to only work within installed web apps.  This aligns with the existing `AccentColor` and `AccentColorText` CSS system color keywords.

Previously, when `accent-color` was set to `auto`, form controls would adopt the system accent color in all contexts (both web pages and installed web apps).  This change ensures that system accent colors are only available in installed web app contexts, which provides more consistent developer and user expectations while adhering to fingerprinting restrictions.

See also:
* [accent-color](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/accent-color) at MDN.

## SVG features


<!-- ------------------------------ -->
#### `path` attribute on SVG `<textPath>` element

The `<textPath>` SVG element now supports a `path` attribute, which lets you define text path geometry inline, by using SVG path data.

Instead of always needing to reference a separate `<path>` element by using the `href` attribute, you can now include the path data directly in the `<textPath>` element's `path` attribute.

When both `path` and `href` are specified, the `path` attribute takes precedence.

See also:
* [&lt;textPath&gt;](https://developer.mozilla.org/docs/Web/SVG/Reference/Element/textPath) at MDN.

## Web APIs

The following new Web API features are included in Microsoft Edge.


<!-- ------------------------------ -->
#### Clear the `window.name` value for cross-site navigations that switch browsing context group

Previously, the value of the `window.name` property was preserved throughout the lifetime of a browser tab, even when a cross-site navigation switched browsing context groups.

The `window.name` property is now cleared when a cross-site navigation switches browsing context groups.

See also:
* [Window: name property](https://developer.mozilla.org/docs/Web/API/Window/name) at MDN.
* [Browsing context](https://developer.mozilla.org/docs/Glossary/Browsing_context) at MDN.


<!-- ------------------------------ -->
#### Device Bound Session Credentials

Device Bound Session Credentials (DBSC) provides a way to bind session credentials to a specific device, making stolen session cookies harder to reuse.

This improves account security for signed-in sessions and helps reduce session hijacking risks.

See also:
* [Device Bound Session Credentials](https://w3c.github.io/webappsec-dbsc/)


<!-- ------------------------------ -->
#### Web App Origin Migration

Web App Origin Migration enables a Progressive Web App (PWA) to migrate to a new origin while preserving trust, install state, and applicable permissions.

This helps teams safely move apps across domains without forcing users to reinstall.

See also:
* [Web App Manifest](https://developer.mozilla.org/docs/Web/Manifest) at MDN.


<!-- ------------------------------ -->
#### OpaqueRange

`OpaqueRange` represents a live text range inside text controls such as `<input>` and `<textarea>`.

Use `OpaqueRange` to enable range-based operations, including geometry queries and integrations with highlighting workflows.

See also:
* [Range](https://developer.mozilla.org/docs/Web/API/Range) at MDN.


<!-- ------------------------------ -->
#### Web App HTML Install Element

The new Web App HTML install element enables declarative install UI for web apps directly in markup.

This allows install prompts to be integrated into page structure without relying only on imperative JavaScript flows.

See also:
* [Progressive web apps (PWAs)](https://developer.mozilla.org/docs/Web/Progressive_web_apps) at MDN.


<!-- ------------------------------ -->
#### Connection allowlists

Connection allowlists let apps declare and enforce explicit allowlists for network endpoints that can be contacted by web platform networking APIs.

This improves outbound network control and reduces the risk of unintended connections.

See also:
* [Fetch API](https://developer.mozilla.org/docs/Web/API/Fetch_API) at MDN.


<!-- ------------------------------ -->
#### Autofill event

The new `autofill` event lets you detect when browser autofill updates form controls.

This makes it easier to adapt custom UI, validation, or dependent form logic after autofill completes.

See also:
* [HTMLElement: input event](https://developer.mozilla.org/docs/Web/API/Element/input_event) at MDN.


<!-- ------------------------------ -->
#### CSSPseudoElement interface

The `CSSPseudoElement` interface represents pseudo-elements in the CSS Object Model.  It is returned by `Element.pseudo(type)` and supports nested pseudo-elements.

This makes it easier to inspect and manipulate pseudo-element-related behavior in advanced styling scenarios.

See also:
* [Pseudo-elements](https://developer.mozilla.org/docs/Web/CSS/Pseudo-elements) at MDN.


<!-- ------------------------------ -->
#### `dropEffect` handling in drag and drop events

When using the Drag and Drop API, the `dropEffect` attribute now has the correct predetermined values during `dragenter`, `dragover`, and `dragleave` events, as specified by the HTML standard.

The `dropEffect` is now based on the current `effectAllowed` for `dragenter` and `dragover` events, and is always `"none"` for `dragleave` events.

See also:
* [HTML Drag and Drop API](https://developer.mozilla.org/docs/Web/API/HTML_Drag_and_Drop_API) at MDN.
* [DataTransfer: dropEffect property](https://developer.mozilla.org/docs/Web/API/DataTransfer/dropEffect) at MDN.


<!-- ------------------------------ -->
#### Element-scoped view transitions

View transitions can now be scoped to individual HTML elements using `element.startViewTransition()`.

When a view transition is scoped to an element, the transition pseudo-elements are affected by ancestor clips and transforms.  Multiple transitions on separate elements can run concurrently, giving you more flexibility in creating complex animations.

See also:
* [View Transitions API](https://developer.mozilla.org/docs/Web/API/View_Transitions_API) at MDN.


<!-- ------------------------------ -->
#### Get Secure Payment Confirmation capabilities

Use the `PaymentRequest.getSecurePaymentConfirmationCapabilities()` static method to determine which Secure Payment Confirmation features are supported in the browser.

This async method returns an object describing the browser's Secure Payment Confirmation capabilities, helping you decide whether to use this feature for your payment flow.

See also:
* [Using Secure Payment Confirmation](https://developer.mozilla.org/docs/Web/API/Payment_Request_API/Using_secure_payment_confirmation) at MDN.


<!-- ------------------------------ -->
#### IDNA ContextJ rules

To improve interoperability with other browsers, Microsoft Edge now implements the IDNA ContextJ rules, by disallowing ZWNJ (U+200C ZERO WIDTH NON-JOINER) and ZWJ (U+200D ZERO WIDTH JOINER) characters in most places within URLs.

IDNA is the mechanism by which non-ASCII characters can occur in domain names.  For example, the URL `http://네이버.한국/` is encoded as `http://xn--950bt9s8xi.xn--3e0b707e/`.

See also:
* [Better Unicode support](./143.md#better-unicode-support) in _Microsoft Edge 143 web platform release notes (Dec. 2025)_.


<!-- ------------------------------ -->
#### JSON and style support for `link rel=modulepreload`

You can now use `<link rel="modulepreload">` to preload JSON and CSS modules in addition to JavaScript modules.

Specify the module type using the `as` attribute:

* `<link rel="modulepreload" as="style" href="...">` for CSS modules.

* `<link rel="modulepreload" as="json" href="...">` for JSON modules.

This brings feature parity across different module types for preloading.

See also:
* [rel="modulepreload"](https://developer.mozilla.org/docs/Web/HTML/Reference/Attributes/rel/modulepreload) at MDN.


<!-- ------------------------------ -->
#### Local Network Access restrictions on Service Worker `WindowClient.navigate()`

Local Network Access restrictions now apply to navigations initiated by Service Workers through `WindowClient.navigate()`.

The `WindowClient` is used as the initiator of the navigation to determine if the navigation is a Local Network Access request.  This only applies if the `WindowClient` being navigated is a subframe; Microsoft Edge does not currently enforce any Local Network Access restrictions on main frame navigations.

See also:
* [WindowClient](https://developer.mozilla.org/docs/Web/API/WindowClient) at MDN.


<!-- ------------------------------ -->
#### Local Network Access restrictions for WebSockets

Local Network Access restrictions now apply to WebSocket connections to local addresses, which will trigger permission prompts.

All current Local Network Access enterprise policies apply to WebSocket restrictions (`LocalNetworkAccessAllowedForUrls`, `LocalNetworkAccessBlockedForUrls`, and `LocalNetworkAccessRestrictionsTemporaryOptOut`).

See also:
* [New permission prompt for Local Network Access](https://developer.chrome.com/blog/local-network-access) on developer.chrome.com.


<!-- ------------------------------ -->
#### Local Network Access restrictions for WebTransport

WebTransport requests to the user's local network are now restricted behind a permission prompt.

A local network request is any request from a public website to a local IP address or loopback, or from a local website (such as an intranet) to loopback.  Gating these requests behind a permission reduces the ability of sites to fingerprint the user's local network.  This permission is restricted to secure contexts.

See also:
* [WebTransport](https://developer.mozilla.org/docs/Web/API/WebTransport) at MDN.


<!-- ------------------------------ -->
#### Manifest localization

Web app manifests now support localization, allowing apps to adapt their names, descriptions, icons, and shortcuts to the user's language and region.

Provide localized values in the web app manifest, and the browser automatically selects the appropriate resources based on the user's language settings.

See also:
* [Web app manifests](https://developer.mozilla.org/docs/Web/Manifest) at MDN.


<!-- ------------------------------ -->
#### `Math.sumPrecise`

The new `Math.sumPrecise()` method returns the sum of values in an iterable using a more precise algorithm than naive summation.

This method helps avoid floating-point precision errors when summing multiple numbers.

See also:
* [Math](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Math) at MDN.


<!-- ------------------------------ -->
#### Pointer event suppression on drag start

When a drag operation starts, the browser now sends `pointercancel`, `pointerout`, and `pointerleave` events to the drag source to indicate that the pointer event stream has ended.

This behavior is now fully implemented across all platforms, ensuring that the drag source doesn't expect any more events from this pointer during the drag operation.

See also:
* [HTML Drag and Drop API](https://developer.mozilla.org/docs/Web/API/HTML_Drag_and_Drop_API) at MDN.
* [Pointer events](https://developer.mozilla.org/docs/Web/API/Pointer_events) at MDN.


<!-- ------------------------------ -->
#### `pseudoTarget` property on events

The `pseudoTarget` property is now available on `UIEvent`, `AnimationEvent`, and `TransitionEvent` objects.

When an event originates from a pseudo-element, `pseudoTarget` returns a `CSSPseudoElement` object representing that pseudo-element.  Otherwise, `pseudoTarget` is `null`.  This provides more specific information about the event origin, while `Event.target` remains unchanged.

See also:
* [Event.target](https://developer.mozilla.org/docs/Web/API/Event/target) at MDN.


<!-- ------------------------------ -->
#### `Request.isReloadNavigation` attribute

The `isReloadNavigation` read-only boolean attribute is now available on the Fetch API's `Request` interface.

This attribute indicates whether the current navigation request was initiated as a user-triggered reload, such as using the refresh button, `location.reload()`, or `history.go(0)`.  This signal is primarily exposed on the `Request` object within a Service Worker's `FetchEvent`.

See also:
* [Request](https://developer.mozilla.org/docs/Web/API/Request) at MDN.


<!-- ------------------------------ -->
#### Respect `autocorrect="off"` for Windows touch keyboard

The Windows touch keyboard now respects the `autocorrect="off"` attribute on editable elements, including `<input>`, `<textarea>`, and `contenteditable` hosts.

Previously, the touch keyboard would autocorrect words even when `autocorrect="off"` was set.  Now, autocorrections are detected and reverted when the focused element has `autocorrect="off"`.

See also:
* [autocorrect](https://developer.mozilla.org/docs/Web/HTML/Reference/Global_attributes/autocorrect) at MDN.


<!-- ------------------------------ -->
#### Device Memory API limits

The Device Memory API now reports updated values that better reflect modern device capabilities:

* Android devices: 1, 2, 4, or 8 GB

* Other platforms: 2, 4, 8, 16, or 32 GB

These new values replace the outdated values (0.25, 0.5, 1, 2, 4, 8 GB).  This reduces fingerprinting risk, and provides better segmentation for high-end devices.

See also:
* [Device Memory API](https://developer.mozilla.org/docs/Web/API/Device_Memory_API) at MDN.


<!-- ------------------------------ -->
#### Web Serial API on Android

The Web Serial API is now supported on Android, allowing web apps to connect to serial devices through a serial port or removable USB and Bluetooth devices that emulate a serial port.

This is useful for educational, hobbyist, and industrial applications that need to control peripheral devices such as robots, mills, laser cutters, or 3D printers from a connected device.

See also:
* [Web Serial API](https://developer.mozilla.org/docs/Web/API/Web_Serial_API) at MDN.


<!-- ------------------------------ -->
#### Always negotiate data channels in WebRTC

The new `alwaysNegotiateDataChannels` option for `RTCPeerConnection` allows you to negotiate data channels in the SDP offer before creating a data channel.

When enabled, the data channel m-line is placed before any audio or video m-lines and is used as the "offerer-tagged m= section" for BUNDLE.

See also:
* [RTCPeerConnection](https://developer.mozilla.org/docs/Web/API/RTCPeerConnection) at MDN.


<!-- ------------------------------ -->
#### WebXR Layers

WebXR Layers offers a more efficient way of drawing immersive content.

In addition to support for native color and depth textures and texture arrays, it provides support for different layer types that are managed by the system compositor (as opposed to JavaScript).

See also:
* [WebXR Device API](https://developer.mozilla.org/docs/Web/API/WebXR_Device_API) at MDN.


<!-- ------------------------------ -->
#### WebXR Plane Detection

The WebXR Plane Detection API allows sites to retrieve the set of planes detected in the user's environment.

This provides more powerful capabilities than depth-sensing, because planes such as walls that are occluded by objects can still be fully represented if the system is aware of the boundaries of the wall.  Semantic labeling information is also exposed where available, allowing better knowledge about the environment.

See also:
* [WebXR Device API](https://developer.mozilla.org/docs/Web/API/WebXR_Device_API) at MDN.


<!-- ------------------------------ -->
#### XML parsing in Rust for non-XSLT scenarios

Microsoft Edge now uses a Rust-based XML parser in scenarios where XSLT processing is not required.

The Rust XML parser improves security by eliminating memory corruption bugs in XML parsing.  This applies to the DOMParser Web API, `responseXML` of `XMLHttpRequest`, and SVG documents.

See also:
* [DOMParser](https://developer.mozilla.org/docs/Web/API/DOMParser) at MDN.

## Deprecated or removed features


<!-- ------------------------------ -->
#### Remove inline XSLT for production of SVG

Support for using XSLT stylesheets to transform XML data into SVG files has been removed.

Previously, a specially crafted XML file could contain an XSL stylesheet that transforms generic XML data into an SVG file.  This capability is now deprecated and removed as part of the broader XSLT deprecation effort.

#### Fetch retry

Expires on March 24, 2026.

Ensure reliable delivery of critical requests on flaky networks with configurable automatic retries. Set maximum attempts, initial delay, and exponential backoff strategy to recover from transient network errors. For keepalive requests, retries can continue even after page unload, improving data delivery for beacons and analytics.

* [Explainer](https://github.com/explainers-by-googlers/fetch-retry/tree/main)
* [Feedback](https://github.com/whatwg/fetch/issues/1838)
* [Register](https://developer.microsoft.com/en-us/microsoft-edge/origin-trials/trials/1828ff92-dfc0-4cec-879f-4a96010552d8)

#### WebAudio Playout Statistics API

Expires on April 21, 2026.

Detect and measure audio glitches and playout latency in real-time using the `AudioPlayoutStats` interface. Monitor fallback frame events, calculate the fraction of silence due to underperformance, and track average, minimum, and maximum latency to optimize your audio application's user experience and responsiveness.

* [Explainer](https://github.com/WICG/web_audio_playout)
* [Feedback](https://github.com/WICG/proposals/issues/142)
* [Register](https://developer.microsoft.com/en-us/microsoft-edge/origin-trials/trials/63c98ae7-a29e-4dc8-a9a3-d75e9c275045)

#### Writer API

Expires on April 21, 2026.

The Writer API can be used to generate text from a writing task prompt, backed by an on-device AI language model.

Use this API to:
* Generate a textual explanation of structured data.
* Compose a post about a product, based on reviews or a product description.
* Generate a biography based on background information.
* And more.

See also [Summarize, write, and rewrite text with the Writing Assistance APIs](../writing-assistance-apis.md).

* [Explainer](https://github.com/webmachinelearning/writing-assistance-apis/blob/main/README.md#writer-api)

* [Register](https://developer.microsoft.com/en-us/microsoft-edge/origin-trials/trials/e3cf9ae7-e268-49c6-830a-6f86575e9dad)

#### Rewriter API

Expires on April 21, 2026.

The Rewriter API transforms and rephrases input text in requested ways, backed by an on-device AI language model.

Use this API to remove redundancies within input text, in order to:
* Fit the text into a word limit.
* Rephrase the text to suit the intended audience.
* Be more constructive if the text uses inadequate language.

See also [Summarize, write, and rewrite text with the Writing Assistance APIs](../writing-assistance-apis.md).

* [Explainer](https://github.com/webmachinelearning/writing-assistance-apis/blob/main/README.md#rewriter-api)

#### Proofreader API

Expires on May 19, 2026.

The Proofreader API corrects grammar, spelling, and punctuation errors in text by using a small language model (SLM) that is built into Microsoft Edge, from your website's or browser extension's JavaScript code.

See also [Correct grammar and spelling with the Proofreader API](../proofreader-api.md).

#### Prompt API

Expires on June 16, 2026

The Prompt API is an experimental web API that allows you to prompt a small language model (SLM) that is built into Microsoft Edge, from your website's or browser extension's JavaScript code.  Use the Prompt API to generate and analyze text, or to create application logic based on user input, and discover innovative ways to integrate prompt engineering capabilities into your web application.

#### `<usermedia>` HTML element

Expires on July 14, 2026.

The `<usermedia>` HTML element is a browser-controlled element for requesting camera or microphone access.

Using a semantic HTML element instead of JavaScript:
* Provides better clarity to users about the permission request.
* Improves accessibility.
* Prevents manipulative UI patterns.
* Streamlines the workflow, by directly providing the media stream to your application.  This eliminates the need for separate API calls.
