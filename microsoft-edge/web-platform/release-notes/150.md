## CSS features

The following new Cascading Style Sheets (CSS) features are included in Microsoft Edge.


#### `AccentColor` and `AccentColorText` system colors

The `AccentColor` and `AccentColorText` system colors can now be used as CSS color values to access the system accent color that's specified on the user's device.

See also:
* [\<system-color\>](https://developer.mozilla.org/docs/Web/CSS/system-color) at MDN.
* [system Colors](https://drafts.csswg.org/css-color/#css-system-colors) in _CSS Color Module Level 4_.


<!-- ------------------------------ -->
#### Allow optional rounding parameter for `polygon()`

Allows an optional `round` parameter in the `polygon()` CSS shape function.  You can specify a length value to round polygon corners, without manually computing bezier curves.

For example, to create a rounded triangle: `polygon(round 50px, 50% 0%, 0% 100%, 100% 100%)`

See also:
* [CSS polygon() round playground](https://codepen.io/yisi/pen/NPRLEQN) - Interactive demo.
* [[css-shapes] Allow optional rounding parameter for polygon()  #9843](https://github.com/w3c/csswg-drafts/issues/9843) - CSS Working Group issue.


<!-- ------------------------------ -->
#### Animatable zoom

The CSS `zoom` property can be animated, and interpolates as a `<number>`.  You can transition and animate zoom to smoothly scale elements and their layout, complementing existing transform-based scaling.

See also:
* [`zoom` CSS property](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/zoom) at MDN.


<!-- ------------------------------ -->
#### CSS URL request modifiers

The following CSS functions can be used with the CSS `url()` function to control the fetch behavior of the referenced resource:
* `cross-origin()`
* `integrity()`
* `referrer-policy()`

For example, `background-image: url("image.png" cross-origin(anonymous))` fetches the image by using CORS anonymous mode.

This gives you fine-grained control over cross-origin access, subresource integrity, and referrer policy for CSS-loaded resources including:
* Images.
* Fonts.
* SVG references.
* Imported stylesheets.

See also:
* [\<url\> CSS type](https://developer.mozilla.org/docs/Web/CSS/Reference/Values/url_value) at MDN.
* [Request URL Modifiers](https://drafts.csswg.org/css-values-5/#request-url-modifiers) in _CSS Values and Units Module Level 5_.


<!-- ------------------------------ -->
#### CSS `text-fit` property

The CSS `text-fit` property scales the font size of a text node to perfectly fit the width of its containing box.

This property allows you to ensure that headlines or dynamic content fill the available horizontal space, without manual font-size calculations or complex JavaScript workarounds.

This property provides a robust, CSS-native solution for responsive typography that maintains visual alignment across different screen sizes and varying text lengths.

See also:
* [Text fitting: the `text-fit` property](https://drafts.csswg.org/css-text-4/#text-fit-property) in _CSS Text Module Level 4_.


<!-- ------------------------------ -->
#### CSS `background-clip: border-area`

The CSS `background-clip: border-area` property clips an element's background to the area painted by its border strokes, taking `border-width` and `border-style` into account, while ignoring transparency from `border-color`.

This enables gradient borders without `border-image`.

See also:
* [`background-clip` CSS property](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/background-clip) at MDN.


<!-- ------------------------------ -->
#### CSS `image(<color>)` function

The CSS `image()` function enables you to easily generate a solid-color image from any color.

For example, to create a solid red background-image, use `background: image(red);`.

See also:
* [\<image\> CSS type](https://developer.mozilla.org/docs/Web/CSS/Reference/Values/image) at MDN.
* [Image Fallbacks and Annotations: the `image()` notation](https://drafts.csswg.org/css-images-4/#image-notation) in _CSS Images Module Level 4_.


<!-- ------------------------------ -->
#### CSS `light-dark()` with image values

The CSS `light-dark()` function now accepts image values (`url()`, `image-set()`, `none`).

This makes it possible to automatically switch between images based on the user's preferred color scheme.

You can use the `light-dark()` function with image values in the following properties:
* `background-image`
* `list-style-image`
* `border-image-source`
* `cursor`
* `content`

See also:
* [`light-dark()` CSS function](https://developer.mozilla.org/docs/Web/CSS/Reference/Values/color_value/light-dark) at MDN.


<!-- ------------------------------ -->
#### Comma-separated container queries

The `@container` rule now supports multiple container queries, separated by commas.  The `@container` rule applies if at least one of the queries matches.

This makes it possible to have fallback queries for features that aren't supported in all browsers.

Example:

`@container --name1 not-supported(--foo: bar), --name2 (width > 600px) {}`

See also:
* [CSS container querie](https://developer.mozilla.org/docs/Web/CSS/Guides/Containment/Container_queries) at MDN.


<!-- ------------------------------ -->
#### Expose unprintable areas via CSS

Use the CSS `page-margin-safety` property to avoid trying to print to unprintable areas of a sheet of paper.

A printer usually has a small area at each of the four edges of a sheet of paper that the printer is not capable of marking reliably, usually due to the printer's paper handling mechanism.

The default page margins are expected to be bigger than these paper-edge areas, but if you set margins in your app, and want to add `@page` margin boxes (such as for custom headers and footers), the `page-margin-safety` property provides a way of detecting where it's safe to print.

See also:
* [Staying within the printable area: the `page-margin-safety` property](https://drafts.csswg.org/css-page-3/#page-margin-safety) in _CSS Paged Media Module Level 3_.


<!-- ------------------------------ -->
#### Media element pseudo-classes

The following CSS pseudo-classes match `<audio>` and `<video>` elements based on their state:
* `:playing`
* `:paused`
* `:seeking`
* `:buffering`
* `:stalled`
* `:muted`
* `:volume-locked`

See also:
* [`:playing` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:playing) at MDN.
* [`:paused` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:paused) at MDN.
* [`:seeking` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:seeking) at MDN.
* [`:buffering` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:buffering) at MDN.
* [`:stalled` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:stalled) at MDN.
* [`:muted` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:muted) at MDN.
* [`:volume-locked` CSS pseudo-class](https://developer.mozilla.org/docs/Web/CSS/Reference/Selectors/:volume-locked) at MDN.


<!-- ------------------------------ -->
#### CSS `alpha()` function

The CSS `alpha()` function creates a new color by applying a specified alpha value to an existing color.

For example, `alpha(from red / 0.5)` creates a semi-transparent red color with 50% opacity.

See also:
* [Relative Alpha Colors](https://drafts.csswg.org/css-color-5/#relative-alpha) in _CSS Color Module Level 5_.


<!-- ------------------------------ -->
#### `named-feature()` function for CSS `@supports`

The `named-feature()` function allows CSS `@supports` rules to query for a small set of specific named features that are not possible to test for by using other `@supports` mechanisms, but which are valuable to test for.

See also:
* [Extensions to the `@supports` rule](https://drafts.csswg.org/css-conditional-5/#at-supports-ext) in _CSS Conditional Rules Module Level 5_.


<!-- ------------------------------ -->
#### `overscroll-behavior: chain`

The `overscroll-behavior` CSS property now supports the `chain` value.

This property affects two independent effects:
* Scroll propagation.
* Local border effect (such as overscroll stretch).

Previously, `overscroll-behavior` had three values:
* `none`: No propagation, no local effect.
* `auto`: Propagation, local effect.
* `contain`: No propagation, local effect.

This feature adds a new value to complete the set:
* `chain`: Propagation, no local effect.

See also:
* [`overscroll-behavior` CSS property](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/overscroll-behavior) at MDN.
* [Overscroll Behavior Properties](https://drafts.csswg.org/css-overscroll-1/#overscroll-behavior-properties) in _CSS Overscroll Behavior Module Level 1_.


<!-- ------------------------------ -->
#### `flex-wrap:balance`

`flex-wrap:balance` allows you to distribute content between each `flex-line`, so that the layout appears more balanced.

This is similar to `text-wrap:balance`.

See also:
* [`flex-wrap` CSS property](https://developer.mozilla.org/docs/Web/CSS/Reference/Properties/flex-wrap) at MDN.
* [Flex Line Wrapping: the `flex-wrap` property](https://drafts.csswg.org/css-flexbox-2/#flex-wrap-property) in _CSS Flexible Box Layout Module Level 2_.

## HTML features

The following new HTML features are included in Microsoft Edge.


<!-- ------------------------------ -->
#### `focusgroup` attribute


The HTML `focusgroup` attribute standardizes keyboard navigation for composite widgets such as toolbars, tabs, menus, and radio groups.

The `focusgroup` attribute automatically handles the following, without requiring custom JavaScript code:
* The _roving tabindex_ behavior.
* Navigation via arrow keys.
* Focus memory; restores the last-focused element when re-entering the focusgroup.

Example:
```html
<div focusgroup="toolbar wrap" aria-label="Formatting">
  <button>Bold</button>
  <button>Italic</button>
  <button>Underline</button>
</div>
```

See also:
* [Making keyboard navigation effortless](https://blogs.windows.com/msedgedev/2026/03/05/making-keyboard-navigation-effortless/).
* [Request for developer feedback: focusgroup](https://developer.chrome.com/blog/focusgroup-rfc).


<!-- ------------------------------ -->
#### `popover="hint"` attribute behavior changes

The behavior of elements with the `popover="hint"` attribute has changed to improve interactions with `popover="auto"` elements: opening a `popover="hint"` element will no longer inadvertently close unrelated `popover="auto"` elements.

Previously, the interactions between these two types of popover elements could be complex in some situations, such as nesting `popover="auto"` elements inside `popover="hint"` elements.

`popover="hint"` elements are now only hidden when their ancester `popover="auto"` element is hidden, or when a new, unrelated `popover="auto"` is opened.

Additionally, you can now safely nest a `popover="auto"` element inside a `popover="hint"` element.  The nested `popover="auto"` element will behave as a `popover="hint"` element.  This allows use cases such as placing a customizable `<select>` element within a `popover="hint"` element.

See also:
* [Popover API](https://developer.mozilla.org/docs/Web/API/Popover_API) at MDN.
* [`popover` HTML global attribute](https://developer.mozilla.org/docs/Web/HTML/Reference/Global_attributes/popover) at MDN.


<!-- ------------------------------ -->
#### Updating all `<selectedcontent>` element in a customizable `<select>`

When multiple `<selectedcontent>` elements are defined in a customizable `<select>` element, all of them are now be kept up to date with the current selected option, instead of only the first one in DOM order.

See also:
* [Customizable select elements](https://developer.mozilla.org/docs/Learn_web_development/Extensions/Forms/Customizable_select) at MDN.
* [\<selectedcontent\> HTML selected option display element](https://developer.mozilla.org/docs/Web/HTML/Reference/Elements/selectedcontent) at MDN.


<!-- ------------------------------ -->
#### Streaming out-of-order HTML

Use the `<template for>` HTML element and the `<?start>` and `<?end>` processing instruction ranges to update existing parts of the document without needing to use JavaScript.

This feature is part of a bigger initiative to add built-in support for same-document updates on the web.

See also:
* [\<template\> HTML content template element](https://developer.mozilla.org/docs/Web/HTML/Reference/Elements/template) at MDN.
* [Interleaved HTML streaming (patching)](https://github.com/WICG/declarative-partial-updates/blob/main/patching-explainer.md) - Explainer for the broader initiative of which this feature is a part.

## Web APIs

The following new Web API features are included in Microsoft Edge.


<!-- ------------------------------ -->
#### Expose the `FontFaceSet` interface globally

Previously, the `FontFaceSet` interface was not exposed as a global property.  Chromium's interface definition language (IDL) for `FontFaceSet` previously incorrectly applied `LegacyNoInterfaceObject` to `FontFaceSet`, hiding it as a global property.

This change aligns Chromium with the CSS Font Loading spec, and with Safari and Firefox.

See also:
* [FontFaceSet](https://developer.mozilla.org/docs/Web/API/FontFaceSet) at MDN.


<!-- ------------------------------ -->
#### Disable SVG filters on cross-origin iframes and plugins

SVG filters are no longer applied to the following:
* Cross-origin or restricted iframes (such as sandboxed iframes).
* Embedded plugins (such as PDFs).

This prevents potential security issues from cross-origin content being processed through SVG filter effects.

See also:
* [SVG filters](https://developer.mozilla.org/docs/Web/SVG/Guides/SVG_filters) at MDN.
* [`<iframe>` HTML inline frame element](https://developer.mozilla.org/docs/Web/HTML/Reference/Elements/iframe) at MDN.


<!-- ------------------------------ -->
#### `MediaStreamTrackProcessor` frame counters 

The `discardedFrames` and `totalFrames` attributes are now available on the `MediaStreamTrackProcessor` interface.

These counters allow you to monitor the health of your media processing pipeline by tracking:
* The number of frames that were received.
* The number of frames that were dropped due to system resource constraints.

See also:
* [MediaStreamTrackProcessor](https://developer.mozilla.org/docs/Web/API/MediaStreamTrackProcessor) at MDN.


<!-- ------------------------------ -->
#### Opaque origin for `data:` URLs

Dedicated Workers and Shared Workers that are created from `data:` URLs are now assigned a unique opaque origin, instead of inheriting the creator's origin.

This aligns Edge with the HTML Standard and improves security by isolating these workers from same-origin state, such as `BroadcastChannel`, `localStorage`, and `IndexedDB`.

To preserve isolation boundaries, these workers remain in the same storage partition as their creator (for example, by top-level site or nonce).

See also:
* [RFC 2397: The "data" URL scheme](https://datatracker.ietf.org/doc/html/rfc2397).
* [Step 3 of Script settings for workers](https://html.spec.whatwg.org/multipage/workers.html#script-settings-for-workers) in _HTML Living Standard_.


<!-- ------------------------------ -->
#### PWA origin migration

Web App Origin Migration enables a Progressive Web App (PWA) to migrate to a new origin while preserving trust, install state, and applicable permissions.

This helps teams safely move apps across domains without forcing users to reinstall.

See also:
* [Web App Manifest](https://developer.mozilla.org/docs/Web/Manifest) at MDN.


<!-- ------------------------------ -->
#### Get notified when the `scrollBy` and `scrollTo` methods complete

Programmatic scrolling methods, such as `scrollBy` and `scrollTo`, now return a `Promise` object that resolves when the scroll completes.  Use this promise to run code after a smooth-scroll finishes, without relying on a timer or scroll-event polling.

See also:
* [Window: scrollBy() method](https://developer.mozilla.org/docs/Web/API/Window/scrollBy) at MDN.
* [Window: scrollTo() method](https://developer.mozilla.org/docs/Web/API/Window/scrollTo) at MDN.


<!-- ------------------------------ -->
#### Web Speech API: on-device recognition quality

The `SpeechRecognition` interface now supports the `quality` property.

The `quality` property allows you to specify the semantic capability that's required for recognition which uses `processLocally: true`, also known as _on-device recognition_.

The values that are supported by the `quality` property map to increasing task complexity and hardware requirements:
* `command`
* `dictation`
* `conversation`

Using the `quality` property, you can determine if the device can handle your use case by using on-device recognition, or if you should fallback to a cloud recognition service.

See also:
* [SpeechRecognition](https://developer.mozilla.org/docs/Web/API/SpeechRecognition) at MDN.
* [Convert speech to text with the SpeechRecognition API](../speech-recognition-api.md).


#### WebGPU: Immediates


WebGPU now supports immediates.

This feature adds an `immediate` address space in WGSL and `setImmediateData()` methods on render pass, compute pass, and render bundle encoders.  You can use these APIs to pass small, frequently updated values to shaders without creating GPU buffers or bind groups.

This is useful for per-draw data, such as object indices, material indices, and transform values.

See also:
* [WebGPU API](https://developer.mozilla.org/docs/Web/API/WebGPU_API) at MDN.
* [WebGPU](https://gpuweb.github.io/gpuweb/) specification.
* [WebGPU Shading Language (WGSL)](https://gpuweb.github.io/gpuweb/wgsl/) specification.
